Project Overview

Jazz Guitar Trainer は、Berklee College of Music の 7-Position System を完全に実装した ウェブベースの指板ビジュアライザーです。ギタリストがスケール、コードフォーム、ビバップフレーズを 体系的に学習できることを目的としています。

主要機能
  • 18モード対応 — Diatonic 7 + Melodic Minor 7 + Harmonic Minor 2 + Diminished 2
  • 7ポジション生成 — 12キー × 18モード = 216パターンの指板マッピング
  • コード進行モード — 1382曲のJazz Standards インポート、iReal Pro風譜面
  • ビバップフレーズ自動生成 — Charlie Parker スタイルの8音フレーズ
  • Drop 2/3 ボイシング — 20テンプレートによるコードフォーム表示
  • ガイドトーン表示 — 3rd/7th のボイスリーディング可視化
  • BPM自動再生 — メトロノーム、コード音声、フレーズ再生

Position Generation (7-Note Scales)

Berklee 7ポジションシステムの核心は B弦（2弦）が各ポジションで2音のみ という点。 他弦は全て3音。これはB-G弦間の長3度チューニング（他は完全4度）に起因します。

Step 1: Fret Map 生成 — buildFretMap()

6弦すべてについて、フレット0〜22のどこにスケール音があるかをマッピング。

開放弦チューニング (半音値): [4, 11, 7, 2, 9, 4]
                              1E   B   G   D   A   6E

各弦の各フレットについて (openString + fret) % 12 がスケール音に含まれるか判定し、 FretNote = [noteName, fret, semitone] のタプル配列として返します。

Step 2: Position 生成 — generatePositions()

不変ルール
このアルゴリズムは変更禁止。過去の失敗（Greedy matcher, ローテート, offset変更）を経て、 現在の「1:1順次割当」が唯一の正解として確立されています。
  1. B弦ペア取得: ルートペアをスキップし、2度ペアから7つの連続2音ペアを取得
  2. 他弦トリオ列挙: 各弦で最低フレットから3音の連続トリオを昇順に列挙
  3. 1:1 順次割当: trio[i] → pair[i] の単純マッピング
  4. STRING_DEG_OFFSETS を使って各弦の開始度数を決定: { e:3, g:5, d:2, a:6 }
  5. findClosestTrio(): フレット重複に基づきB弦ペアに最も近いトリオを選択 (maxGap=5)

C Ionian Pos 1 リファレンス

1E: F(1), G(3), A(5)     ← 3音
B:  D(3), E(5)           ← 2音 (常に)
G:  A(2), B(4), C(5)     ← 3音
D:  E(2), F(3), G(5)     ← 3音
A:  B(2), C(3), D(5)     ← 3音
6E: F(1), G(3), A(5)     ← 3音

Diminished Positions (8-Note Scales)

ディミニッシュ (W-H / H-W) は対称スケールのため 4ポジション × 2フェーズ = 8ポジション を生成。 dim7 の構成音 (Root, ♭3, ♭5, 6) は短3度間隔で等分布。

  • 各ポジションは5フレット幅、3フレット間隔
  • ポジション順: Pos 1=Root, Pos 2=♭3, Pos 3=♭5, Pos 4=6
  • 6E弦のみ4音、他弦は3音

Phrase Generator — Overview

フレーズジェネレーターは、ビバップスタイルの8分音符8音フレーズを指板座標ベースで生成する アルゴリズムです。Charlie Parker のソロラインを模倣するため、複数のレイヤーが協調して動作します。

設計哲学
「理論的に正しいフレーズ」よりも「ギター指板上で実際に弾けるフレーズ」を優先。 弦距離・フレット距離に基づく物理的制約がスコアリングの中核にあります。

Step 1-2: Note Pool & Instance Scope

buildNotePool()

ポジション内の全スケール音を収集し、オプションでクロマチックアプローチ音を追加します。

PoolNote の構造
{
  noteName: string,       // "C", "D♭", etc.
  stringIdx: number,      // 0 (1E) ~ 5 (6E)
  fret: number,           // フレット番号
  semitone: number,       // 0-11 のピッチクラス

  isApproach: boolean,    // クロマチックアプローチ音

Instance Scope Restriction

音域をコンパクトに保つため、第1インスタンスの範囲に制限します。

  • activePool: instFretMin - 1 〜 instFretMax + 1 のノートのみ
  • activeCtPool: activePool のうちコードトーンのみ
  • プール不足時 (≤6音 or CT<2) は全プールにフォールバック

Step 4: Goal Note Selection

フレーズの最終音（beat 8）を最初に決定します。これが全体の方向性を決めます。

設計根拠
  • 70% の理由: パーカーの実際のソロでは ii-V-I フレーズ境界で 60-80% 程度の 7th→3rd 半音解決が観察される。100% だとフレーズが画一化するため 30% の変化余地を残す。
  • 半音（1 semitone）のみ: diff === 1 || diff === 11 で厳密に半音差のみチェック。全音は含まない。ii→V、V→I のような典型進行でのみ発動し、半音解決がない進行ではフォールバックが機能する。
  • 通常モードのゴール音: 次コードの 3rd そのものではなく、STRONG_RESOLUTION_DEGREE_IDX で推定した解決先の度数に最も近い現コードの CT（特に 3rd/7th）をゴールにする。ビバップでは現コード内で解決方向を示唆するのが定石。

chooseGoalNote() の決定ロジック

進行モード（nextChordContext あり）
  1. 現コードの7thが次コードの3rdに半音で解決する場合 → 70%の確率で現コードの7thをゴールに
  2. 次コードの3rdと1半音差のCTを優先検索
  3. フォールバック: 次コードの3rdと同名のCTを選択
通常モード

STRONG_RESOLUTION_DEGREE_IDX で「このモードが解決する先のコードの3rd」を推定。実際のゴール音は現コードのCTから選択:

ModeResolutionTarget
IonianI → IVIVの3rd (scale deg 6)
Dorianii → VVの3rd (scale deg 7)
MixolydianV → IIの3rd (scale deg 3)
Aeolianvi → iiiiの3rd (scale deg 4)

Step 5: Harmonic Skeleton (CT Skeleton)

フレーズの骨格を形成する最も重要なステップ。強拍 (beats 1, 3, 5, 8) に配置するコードトーンを 事前に計画し、フレーズ全体のハーモニック構造を確保します。

Arpeggio Patterns (9パターン)

コードトーンの並び順を定義。各パターンには方向性（ascending/descending/mixed）があります。

設計根拠: 9パターンの選定
4つのCT (R, 3rd, 5th, 7th) の順列は 4!=24 通り存在するが、音楽的に意味のある配列に絞り込んだ。 3方向 (ascending / descending / mixed) × 各3ローテーション = 9パターン。 上行は R→7th に向かう連続的な上昇、下行はその逆、mixed は跳躍を含むジグザグ配列。 これらは実際のビバップソロで最も頻出するアルペジオ輪郭をカバーする。

各強拍について、skeleton で決められたCTインデックスを 物理的な指板位置に解決 します。 近接クラスター（ピッチ差 ≤2st）を避けつつ、最もピッチ的に近い候補を選択します。

Step 6: Digital Patterns

Charlie Parker の語彙から抽出した 33個のデジタルパターン（音列テンプレート）。 強拍を起点に一括でコミットされ、パーカー的なフレージングを実現します。

発火条件
42%の確率で発火。1フレーズにつき最大1回。強拍CT一致 + 物理的到達可能性をチェック。
設計根拠
  • 42% の発火率: 品質監査テスト (N=5000) で検証済みの経験値。高すぎるとフレーズが画一的になり、低すぎるとパーカー語彙が不足する。33パターンへの拡充により同一パターン出現頻度が下がったため、21パターン時代の35%から引き上げた。
  • デジタルパターン ≒ リック: パーカーのソロから抽出した定型フレーズ（リック）をスケール度数ベースのテンプレートに変換したもの。キーやポジションに依存しない汎用性を持つ。
  • 方向保証: pickByDirection() がパターンの dir (asc/desc) に応じて候補をフィルタリングし、上行パターンでは現在位置より高い音、下行では低い音のみを選択。mixed はフィルタなし。

  • findScaleDegreeNeighbor() — スケール度数オフセットで隣接音を検索

Approach Patterns

ビバップ語法の核心。ターゲット音（主にコードトーン）への クロマチック/ダイアトニックな接近 を 弱拍に挿入し、ジャズらしい推進力を生み出します。

設計根拠: 3rd/7th 優先ターゲティング
アプローチのターゲットとして 3rd と 7th を優先するのはガイドトーン理論に基づく。 3rd はコードのメジャー/マイナーを、7th はコードクオリティ（maj7/dom7/m7）を定義する最重要音。 これらの音へのクロマチックアプローチはビバップ語法の核心であり、コードの機能を強調しつつ 半音の緊張→解決の推進力を生み出す。tryApproachCommitment() 内で近傍CTを探索する際、 3rd/7th に +12 のスコアボーナスを付与して優先度を高めている。

5種類のアプローチパターン

1. Single Below (半音下からの接近)

ターゲットの1フレット下からの単音アプローチ。最もシンプルで頻出。

Pattern: [target - 1 fret] → TARGET
Example: F# → G (G is the target CT)
2. Single Above (半音上からの接近)

ターゲットの1フレット上からの単音アプローチ。

Pattern: [target + 1 fret] → TARGET
Example: A♭ → G
3. Enclosure (エンクロージャー)

ターゲットを上下から挟み込む2音アプローチ。上は最寄りのダイアトニック音、下は半音。

Pattern: [diatonic above] → [chromatic below] → TARGET
Example: A → F# → G
4. Parker Enclosure (パーカー・エンクロージャー)

Charlie Parker が多用した3音エンクロージャー。半音上→2フレット下→半音下。

Pattern: [target + 1] → [target - 2] → [target - 1] → TARGET
Example: A♭ → F → F# → G
5. ♭9 Arpeggio (♭9アルペジオ)

Dom7コード専用の4音アプローチ。ルートの♭9thから始まるdiminishedアルペジオ。

Pattern: [♭9] → [3rd] → [5th] → [♭7th]
Example (G7): A♭ → B → D → F → (next chord's CT)

Scoring System

フレーズ生成の心臓部。各候補ノートに対して 多次元のスコアリング を行い、 重み付きランダム選択 (pickWeighted) で次の音を決定します。

選択方式
最高スコアの音を常に選ぶのではなく、スコアを確率の重みとして使用。 これにより決定論的すぎないバリエーション豊かなフレーズが生成されます。

  1. ビート順ソート: notes を beatPosition 順に整列
