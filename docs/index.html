Project Overview

Jazz Guitar Trainer は、Berklee College of Music の 7-Position System を完全に実装した ウェブベースの指板ビジュアライザーです。ギタリストがスケール、コードフォーム、ビバップフレーズを 体系的に学習できることを目的としています。

主要機能
  • 18モード対応 — Diatonic 7 + Melodic Minor 7 + Harmonic Minor 2 + Diminished 2
  • 7ポジション生成 — 12キー × 18モード = 216パターンの指板マッピング
  • コード進行モード — 1382曲のJazz Standards インポート、iReal Pro風譜面
  • ビバップフレーズ自動生成 — ルールベースエンジンによるビバップフレーズ生成
  • Drop 2/3 ボイシング — 20テンプレートによるコードフォーム表示
  • ガイドトーン表示 — 3rd/7th のボイスリーディング可視化
  • BPM自動再生 — メトロノーム、コード音声、フレーズ再生 (両モード共通)

Position Generation (7-Note Scales)

Berklee 7ポジションシステムの核心は B弦（2弦）が各ポジションで2音のみ という点。 他弦は全て3音。これはB-G弦間の長3度チューニング（他は完全4度）に起因します。

Step 1: Fret Map 生成 — buildFretMap()

6弦すべてについて、フレット0〜22のどこにスケール音があるかをマッピング。

開放弦チューニング (半音値): [4, 11, 7, 2, 9, 4]
                              1E   B   G   D   A   6E

各弦の各フレットについて (openString + fret) % 12 がスケール音に含まれるか判定し、 FretNote = [noteName, fret, semitone] のタプル配列として返します。

Step 2: Position 生成 — generatePositions()

不変ルール
このアルゴリズムは変更禁止。過去の失敗（Greedy matcher, ローテート, offset変更）を経て、 現在の「1:1順次割当」が唯一の正解として確立されています。
  1. B弦ペア取得: ルートペアをスキップし、2度ペアから7つの連続2音ペアを取得
  2. 他弦トリオ列挙: 各弦で最低フレットから3音の連続トリオを昇順に列挙
  3. 1:1 順次割当: trio[i] → pair[i] の単純マッピング
  4. STRING_DEG_OFFSETS を使って各弦の開始度数を決定: { e:3, g:5, d:2, a:6 }
  5. findClosestTrio(): フレット重複に基づきB弦ペアに最も近いトリオを選択 (maxGap=5)

C Ionian Pos 1 リファレンス

1E: F(1), G(3), A(5)     ← 3音
B:  D(3), E(5)           ← 2音 (常に)
G:  A(2), B(4), C(5)     ← 3音
D:  E(2), F(3), G(5)     ← 3音
A:  B(2), C(3), D(5)     ← 3音
6E: F(1), G(3), A(5)     ← 3音

Diminished Positions (8-Note Scales)

ディミニッシュ (W-H / H-W) は対称スケールのため 4ポジション × 2フェーズ = 8ポジション を生成。 dim7 の構成音 (Root, ♭3, ♭5, 6) は短3度間隔で等分布。

  • 各ポジションは5フレット幅、3フレット間隔
  • ポジション順: Pos 1=Root, Pos 2=♭3, Pos 3=♭5, Pos 4=6
  • 6E弦のみ4音、他弦は3音

Phrase Generator — Overview

フレーズジェネレーターは、ルールベースアーキテクチャでビバップフレーズを生成します。 bebop-construction-rules.md のルールカタログに基づき、指板空間 (PoolNote[]) で直接フレーズを組み立てます。 ビバップスケール、セグメント関数、テンプレート、Musical Forces スコアリングを組み合わせ、 音楽理論に基づいた自然なビバップラインを生成します。生成失敗時は null を返却します。

機能

  • ルールベース生成: 11種のテンプレートと9種のセグメント関数でフレーズを構築
  • 可変リズム: 4分音符(q)、3連符(t)、8分音符(e)、16分音符(s)の混合に対応
  • 拍数選択: 通常モードで2/3/4拍のフレーズ長を選択可能
  • ゴールノート選択: 指板クリックでゴール音を指定し、フレーズの到達点を制御
  • ピアノロール: PhraseAnalysisPanel に時間×ピッチのSVG可視化を追加
  • セグメント境界表示: フレーズ内の各セグメント境界を分析テーブルに表示。ピアノロールにも境界の垂直破線を描画
設計哲学
ビバップ構造ルール (David Baker, Hal Galper, Anton Schwartz 等の教育的コンセンサス) に基づき、 テンプレートとセグメント関数の組み合わせで音楽的なフレーズを構築。 Musical Forces (Steve Larson) による重力・磁力・慣性のスコアリングで自然な音選択を実現。

アーキテクチャ

  • ビバップスケール (bebopScales.ts): Dominant / Major / Dorian / Harmonic Minor の4種。モード→ビバップスケールマッピング。Diatonic 7 + Lydian Dom + Altered の9モード対応
  • セグメント関数 (bebopSegments.ts): Arpeggio, ScaleRun (表拍GT優先), Enclosure (4タイプ+Delayed Resolution), 1-2-3-5, dim7From3rd, UpperStructure, ApproachCT (6タイプ+Musical Forces), Chromatic, OctaveDisp (Honeysuckle Rose) の9種。全て PoolNote[] → PoolNote[] で指板空間内動作。beatParity対応
  • テンプレート (bebopTemplates.ts): 11個のフレーズ構造テンプレート (Arp↑+Scale↓, Scale↓, Honeysuckle 等)。品質フィルタ・コンター親和 (+8)・重み付きランダム選択
  • スケジューラー (bebopScheduler.ts): テンプレートの各セグメントを実行し、セグメント接合部の同ピッチ重複を自動除去。多セグメントテンプレートの接合部で15%確率で休符/ブレスを挿入 (70%裏拍開始)。assignRhythms() でセグメントタイプ別に可変リズムを割当 (§10準拠: アルペジオ→3連符25% (第1セグメント裏拍開始も許可), エンクロージャー→16分35% (≥3音), クロマチック→16分40% (≥2音), スケールラン→8分固定, 最終CT→4分18%)。CT表拍プロモーション (非CTスケール音を±2半音・±1弦以内のCTにスワップ)。テンション終止 (CT終止の26%を隣接非CTスケール音にスワップ→CT終止率~70%)。拍位置ベース品質チェック (CT表拍配置率≥40%, GT強拍ソフトチェック, 音域4-15半音, 跳躍≤9半音, 方向転換裏拍チェック) を適用
  • Musical Forces (§9 Steve Larson): bestCandidate() に重力 (+5/-3)・磁力 (+5)・慣性 (+3) のスコアリングを加算。±4スコアジッター (stepwise/approach選択時のみ) でピッチ列多様性を確保
  • エントリポイント (bebopGenerator.ts): generatePhraseRule() — buildNotePool → コンター選択 (重み付き: arch33/desc28/wave18/rev-arch13/asc8) → beatOffset選択 (70%裏拍) → GT優先開始音 → スケジューラー実行 → 5リトライ → フォールバック

bebop-construction-rules.md 準拠改善

  • アプローチノート多様化 (§3): segApproachCT が6タイプのアプローチを WJD 統計ベースの重みで選択。 chrom-below (w=22), diatonic-above (w=25), chrom-above (w=16), diatonic-below (w=12), dbl-chrom-below (w=8), dbl-chrom-above (w=7)。§3 (Anton Schwartz): 上方アプローチはdiatonic優先、chromatic は慎重に使用。Musical Forces (重力/磁力/慣性) でCTターゲット選択を最適化。
  • エンクロージャー4タイプ化 (§4): Mixed (w=40), Diatonic (w=25), Chromatic (w=20), 3-note (w=15) の重み付き選択。 Delayed Resolution (30%) でターゲットを裏拍にずらすバリエーションも生成。
  • CT終止改善 (§6): スケジューラーの品質チェック後に最終音のCTスワップを試行。 ±3半音・±1弦以内の最近接CTが見つかればスワップ、見つからなければそのまま通過 (reject しない)。 さらにCT終止の26%を隣接非CTスケール音にスワップ (テンション終止) し、CT終止率~70%を実現 (WJD 63.7%に近接)。
  • 休符/ブレス挿入 (§10): 多セグメントテンプレートのセグメント接合部で15%確率で8分/4分休符を挿入。 休符後の次セグメント開始は70%裏拍/30%表拍 (フレーズ開始と同じ確率)。全フレーズの約11%に休符が出現。 オーディオ・SVG・ピアノロール・フレーズ分析すべてで休符に対応。
  • 開始拍分布 (§6): startHint なし (単体生成) 時に 70% で裏拍 (beatOffset=0.5) 開始。 接続フレーズ (startHint あり) ではギャップなし (beatOffset=0)。 beatParity をセグメント関数に伝播し、ScaleRun/Enclosure のビバップパッシングトーン/CT 表拍判定を正確化。
  • 強拍ガイドトーン優先 (§1): 開始音は 3rd/7th に2倍の重み。スケールラン中も表拍でGT候補があれば優先選択。品質チェックで強拍 (beat 1, 3) が2つ以上ある場合にGTゼロならソフト reject。
  • 方向転換裏拍チェック (§9): 方向転換の60%超が表拍 (各拍頭) で発生するフレーズを reject。Barry Harris ルール準拠。
  • コンター選択 (§5): 重み付き選択 — arch (33), descending (28), wave (18), reverse-arch (13), ascending (8)。均等25%から修正。

Note Pool & Instance Scope

buildNotePool()

ポジション内の全スケール音を収集し、オプションでクロマチックアプローチ音を追加します。

PoolNote の構造
{
  noteName: string,       // "C", "D♭", etc.
  stringIdx: number,      // 0 (1E) ~ 5 (6E)
  fret: number,           // フレット番号
  semitone: number,       // 0-11 のピッチクラス

Instance Scope Restriction

音域をコンパクトに保つため、第1インスタンスの範囲に制限します。

  • activePool: instFretMin - 1 〜 instFretMax + 1 のノートのみ
  • activeCtPool: activePool のうちコードトーンのみ
  • プール不足時 (≤6音 or CT<2) は全プールにフォールバック

Goal Note Selection

フレーズの最終音（beat 8）を最初に決定します。これが全体の方向性を決めます。

設計根拠
  • 70% の理由: パーカーの実際のソロでは ii-V-I フレーズ境界で 60-80% 程度の 7th→3rd 半音解決が観察される。100% だとフレーズが画一化するため 30% の変化余地を残す。
  • 半音（1 semitone）のみ: diff === 1 || diff === 11 で厳密に半音差のみチェック。全音は含まない。ii→V、V→I のような典型進行でのみ発動し、半音解決がない進行ではフォールバックが機能する。
  • 通常モードのゴール音: 次コードの 3rd そのものではなく、STRONG_RESOLUTION_DEGREE_IDX で推定した解決先の度数に最も近い現コードの CT（特に 3rd/7th）をゴールにする。ビバップでは現コード内で解決方向を示唆するのが定石。

chooseGoalNote() の決定ロジック

進行モード（nextChordContext あり）
  1. 現コードの7thが次コードの3rdに半音で解決する場合 → 70%の確率で現コードの7thをゴールに
  2. 次コードの3rdと1半音差のCTを優先検索
  3. フォールバック: 次コードの3rdと同名のCTを選択
通常モード

STRONG_RESOLUTION_DEGREE_IDX で「このモードが解決する先のコードの3rd」を推定。実際のゴール音は現コードのCTから選択:

ModeResolutionTarget
IonianI → IVIVの3rd (scale deg 6)
Dorianii → VVの3rd (scale deg 7)
MixolydianV → IIの3rd (scale deg 3)
Aeolianvi → iiiiの3rd (scale deg 4)

Chord Forms (Drop 2 / Drop 3 Voicings)

各ポジション内で弾ける Drop 2 / Drop 3 ボイシングを自動検出し、指板上にハイライト表示。

ボイシング理論

  • Drop 2: 4-way close voicing の上から2番目の音を1オクターブ下げる → 4連続弦
  • Drop 3: 上から3番目を1オクターブ下げる → 1弦スキップの4弦
  • 転回形はベース音で命名（ジャズ標準）: Root, 1st (3rdベース), 2nd (5thベース), 3rd (7thベース)

Audio Engine

すべて Web Audio API ベース。外部ライブラリ依存なし。

  • Karplus-Strong (Guitar) — 物理モデリングによるギター単音シンセ (playKSNote()): ノイズ励起 → 6-pass移動平均 → KSループ (0.4/0.6 LP + decay 0.996) → LP 2200Hz + HighShelf -6dB
  • Sawtooth + Formant (Saxophone) — リード楽器シンセ (playSaxNote()): Sawtooth発振 → 並列バンドパス (F1=500Hz body, F2=1400Hz reed) → LP warmth → Sine LFO 5.5Hz ビブラート (150ms遅延)
  • Electric Piano (Chord) — エレピシンセ (playEPNote()): Sine基音 + 2nd倍音 (ウォーム減衰) + 3rd倍音 (アタックのみ) → LP 4500Hz。コードストラムに使用
  • Instrument Dispatcher — playNote(ctx, freq, vol, time, dur, instrument) でフレーズ楽器別にディスパッチ (guitar/saxophone)
  • Chord Strum — ボイシングの同時/アルペジオ発音 (playChordStrum()) — ローズピアノ音
  • Click — メトロノーム用オシレーター (playClick())
  • Phrase Scheduling — schedulePhrase() で Web Audio API タイムスタンプに基づく正確な発音 (instrument パラメータ対応)

Chart Layout (iReal Pro Style)

Data Flow

JazzStandards.json ↓ fetchJazzStandards() RawJazzStandard { Sections: RawSection[] } ↓ extractStructuredChords() StructuredSection { measures: MeasureChord[][], endings?, repeats? } ↓ songToProgression() Progression { chords: ChordSlot[], chartLayout: ChartLayout } ↓ ChordChart component CSS Grid 描画

Design Decisions

  • chords: ChordSlot[] フラット配列 = ロジックのsingle source of truth
  • chartLayout?: ChartLayout = 表示用メタデータ（chords[] へのインデックス参照）
  • ユーザー作成進行（layoutなし）→ deriveChartLayout() で1コード/小節に自動生成
  • beatWidths: フレックス比率 — ChordChart で flex: beats, BPM 再生で (bw/bwSum)*4

Phrase Path Rendering

PhrasePath.tsx — Catmull-Rom スプラインを Cubic Bezier に変換して SVG <path> で描画。

  • グラデーション: per-beat (#FFA0B0 → #BBA0FF)
  • ノート形状:
    • CT = 塗り潰し円
    • アプローチ = ダイヤモンド
    • スケールトーン = 枠円（ストロークのみ）
  • 拍番号: 各ノートの横に表示
  • アニメーション: 各ノートのフェードインは beatStart ベースのタイミング（リズムに忠実な表示）
  • 視認性改善:
    • Crowding-aware routing: 各セグメントの中点近接度を距離減衰で計測し、密集エリアの線を垂直方向にファンアウト（最大3層）
    • タンジェント抑制: 短セグメント（同フレット弦移動等）は Catmull-Rom タンジェントをセグメント長比例で減衰させ、ほぼ直線に
    • 折り返しbow抑制: 近接フレット間の方向反転でのbow量をセグメント長に比例してスケール
    • 再訪問マーカー: 75%縮小 + 次ノート方向に6pxオフセット
    • ストローク幅テーパリング: 始点2.2px → 終点1.4pxで時間方向を明示
    • 開始ノート強調: マーカー1.2倍 + 半透明リング

Core Types (展開) 
// 指板上の音
type FretNote = [noteName: string, fret: number, semitone: number]

  id: number

  range: [min: number, max: number]
  instances: PositionInstance[]
}

// モード
interface Mode {
  key: string              // "ionian", "dorian", etc.
  name: string             // Display name
  semi: number[]           // Semitone pattern [0,2,4,5,7,9,11]
  notes: string[]          // Note names ["C","D","E",...]
  degrees: string[]        // Degree labels
  chord: string            // Chord symbol
  chordQuality: string     // "maj7", "m7", "7", etc.
  chordTones: string[]     // [R, 3rd, 5th, 7th]
  chordSub?: string        // Subscript suffix
}

// フレーズノート
interface PhraseNote {
  noteName: string
  stringIdx: number        // 0-5
  fret: number
  semitone: number         // 0-11
  isChordTone: boolean
  isApproach: boolean
  isExtension?: boolean
  beatPosition: number     // 1-8
  isStrong: boolean        // true = 強拍 (metrically strong: beat 1, 3)
  approachGroup?: ApproachGroupInfo
  digitalPattern?: DigitalPatternTag   // デジタルパターンのタグ
  isDim7Tone?: boolean                 // dim7アルペジオ構成音 (♭9等)
  isBebopPassing?: boolean             // ビバップ経過音
  isSkeletonBeat?: boolean             // スケルトン拍
  isRest?: boolean                     // 休符 (セグメント接合部ブレス)
  duration?: RhythmType                // 音価 (q/t/e/s)
  beatStart?: number                   // 絶対拍位置 (0起点)
}

  name: string             // "1-2-3-5"
  position: number         // 0-based index
  size: number             // pattern length
}
