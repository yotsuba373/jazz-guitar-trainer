Project Overview

Jazz Guitar Trainer は、Berklee College of Music の 7-Position System を完全に実装した ウェブベースの指板ビジュアライザーです。ギタリストがスケール、コードフォーム、ビバップフレーズを 体系的に学習できることを目的としています。

主要機能
  • 18モード対応 — Diatonic 7 + Melodic Minor 7 + Harmonic Minor 2 + Diminished 2
  • 7ポジション生成 — 12キー × 18モード = 216パターンの指板マッピング
  • 練習モード — 1382曲のJazz Standards インポート、iReal Pro風譜面
  • リック練習 — リックDB (MIDI録音) からコード品質に合うリックを選択・再生・練習
  • Drop 2/3 ボイシング — 20テンプレートによるコードフォーム表示
  • ガイドトーン表示 — 3rd/7th のボイスリーディング可視化
  • BPM自動再生 — メトロノーム (再生時自動、音量>0で有効)、コード音声、フレーズ再生 (両モード共通)
  • リックお気に入り — ★ボタンでお気に入り登録、リスト上位にソート、localStorage永続化
  • 曲ごとのBPM保存 — 各進行に BPM を保存・復元、曲切替時に自動適用
  • スウィングモード — タイミング+ダイナミクス+アーティキュレーションの多次元スウィング
  • カウントイン — 再生開始前に1-2小節クリック、サイクル切替 (2小節→OFF→1小節→2小節)、音量調節可、アウフタクト対応
  • 小節ループ — コード譜面上で小節を選択してループ再生 (単一/範囲)、オレンジ下線ハイライト
  • 表示倍率スライダー — 画面右下固定、CSS zoom で100-150% (1%刻み)、リセットボタン付き
  • ドラムパターン — Hydrogen GM / カスタム WAV ドラム (ride/hihat/kick/snare)。MIDI パターン DB 対応 (DAW 録音→パーサー→JSON→ランダム選択)、フォールバック時は iReal Pro 風アルゴリズム生成 (フェザリング、ゴースト、コンピング、ヒューマナイゼーション)、スウィング対応、メトロノームとの切替

Position Generation (7-Note Scales)

Berklee 7ポジションシステムの核心は B弦（2弦）が各ポジションで2音のみ という点。 他弦は全て3音。これはB-G弦間の長3度チューニング（他は完全4度）に起因します。

Step 1: Fret Map 生成 — buildFretMap()

6弦すべてについて、フレット0〜22のどこにスケール音があるかをマッピング。

開放弦チューニング (半音値): [4, 11, 7, 2, 9, 4]
                              1E   B   G   D   A   6E

各弦の各フレットについて (openString + fret) % 12 がスケール音に含まれるか判定し、 FretNote = [noteName, fret, semitone] のタプル配列として返します。

Step 2: Position 生成 — generatePositions()

不変ルール
このアルゴリズムは変更禁止。過去の失敗（Greedy matcher, ローテート, offset変更）を経て、 現在の「1:1順次割当」が唯一の正解として確立されています。
  1. B弦ペア取得: ルートペアをスキップし、2度ペアから7つの連続2音ペアを取得
  2. 他弦トリオ列挙: 各弦で最低フレットから3音の連続トリオを昇順に列挙
  3. 1:1 順次割当: trio[i] → pair[i] の単純マッピング
  4. STRING_DEG_OFFSETS を使って各弦の開始度数を決定: { e:3, g:5, d:2, a:6 }
  5. findClosestTrio(): フレット重複に基づきB弦ペアに最も近いトリオを選択 (maxGap=5)

C Ionian Pos 1 リファレンス

1E: F(1), G(3), A(5)     ← 3音
B:  D(3), E(5)           ← 2音 (常に)
G:  A(2), B(4), C(5)     ← 3音
D:  E(2), F(3), G(5)     ← 3音
A:  B(2), C(3), D(5)     ← 3音
6E: F(1), G(3), A(5)     ← 3音

Diminished Positions (8-Note Scales)

ディミニッシュ (W-H / H-W) は対称スケールのため 4ポジション × 2フェーズ = 8ポジション を生成。 dim7 の構成音 (Root, ♭3, ♭5, 6) は短3度間隔で等分布。

  • 各ポジションは5フレット幅、3フレット間隔
  • ポジション順: Pos 1=Root, Pos 2=♭3, Pos 3=♭5, Pos 4=6
  • 6E弦のみ4音、他弦は3音

Chord Forms (Drop 2 / Drop 3 Voicings)

各ポジション内で弾ける Drop 2 / Drop 3 ボイシングを自動検出し、指板上にハイライト表示。

ボイシング理論

  • Drop 2: 4-way close voicing の上から2番目の音を1オクターブ下げる → 4連続弦
  • Drop 3: 上から3番目を1オクターブ下げる → 1弦スキップの4弦
  • 転回形はベース音で命名（ジャズ標準）: Root, 1st (3rdベース), 2nd (5thベース), 3rd (7thベース)

Audio Engine

すべて Web Audio API ベース。外部ライブラリ依存なし。

  • Karplus-Strong (Guitar) — 物理モデリングによるギター単音シンセ (playKSNote()): ノイズ励起 → 6-pass移動平均 → KSループ (0.4/0.6 LP + decay 0.996) → LP 2200Hz + HighShelf -6dB
  • Sawtooth + Formant (Saxophone) — リード楽器シンセ (playSaxNote()): Sawtooth発振 → 並列バンドパス (F1=500Hz body, F2=1400Hz reed) → LP warmth → Sine LFO 5.5Hz ビブラート (150ms遅延)
  • Electric Piano (Chord, fallback) — エレピシンセ (playEPNote()): Sine基音 + 2nd倍音 (ウォーム減衰) + 3rd倍音 (アタックのみ) → LP 4500Hz。smplr ピアノ未ロード時のフォールバック
  • SoundFont Piano (Chord) — smplr ライブラリ (playSmplrPianoComp()): acoustic_grand_piano SoundFont サンプルによるリアルなピアノ音。ジャズピアノボイシング (LH: Root+5th, RH: 3rd+7th シェルボイシング) を buildJazzPianoVoicing() で自動生成。初回アクセス時に非同期ロード、ロード完了まで再生ボタンにスピナー表示。stopId でコードごとに voice グループを分離し、同一 MIDI ノートの連続でも音欠けしない。個別 stop 関数も保持し、look-ahead で事前スケジュール済みのノートも停止時に確実にキャンセル
  • SoundFont Bass (Walking Bass) — smplr ライブラリ (playSmplrBassLine()): acoustic_bass SoundFont でウォーキングベースライン再生。generateBassLine() がコード品質・拍数・次コードルートから自動生成 (1拍=ルートのみ、2拍=ルート+アプローチ、3-4拍=ルート→3rd/5th→5th/8va→半音アプローチ)。音域 E1(28)~G3(55)。ピアノと同じ stopId + 個別 stop 関数パターンで管理
  • Instrument Dispatcher — playNote(ctx, freq, vol, time, dur, instrument) でフレーズ楽器別にディスパッチ (guitar/saxophone)
  • Chord Strum — コード発音 — 練習モード: smplr ジャズピアノコンピング (playSmplrPianoComp()、LH Root+5th / RH 3rd+7th)、辞典モード/フォールバック: エレピシンセ (playChordStrum())
  • Click — メトロノーム用オシレーター (playClick())
  • Phrase Scheduling — schedulePhrase() で Web Audio API タイムスタンプに基づく正確な発音 (instrument パラメータ対応)

Count-In (カウントイン)

再生開始前にメトロノームクリックを鳴らし、テンポを体に入れてから演奏を開始できる機能。

Measure Loop (小節ループ)

苦手な箇所を繰り返し練習するための小節単位ループ機能。

  • コード譜面の各小節右上にループアイコン (ホバー時表示) → クリックでループ設定
  • 1回目クリック: 単一小節ループ。別の小節をクリック: 範囲拡張 (min-max)。同じ小節再クリック: 解除
  • ループ範囲内の小節: オレンジ下線ハイライト。範囲外: dim 表示 (opacity 0.35)
  • 再生中、ループ範囲の最後のコード完了後に範囲先頭へ自動的に戻る
  • GlobalAudioControls にループ状態ラベル + 解除ボタン表示
  • 曲ごとに Progression.loopRange として localStorage に永続化。進行切替時に復元
  • 実装: buildPlaybackSeq() の各エントリに measureFlatIdx を付与し、auto-advance effect で次コードのインデックスがループ範囲外ならループ先頭に戻す

Chart Layout (iReal Pro Style)

Data Flow

JazzStandards.json ↓ fetchJazzStandards() RawJazzStandard { Sections: RawSection[] } ↓ extractStructuredChords() StructuredSection { measures: MeasureChord[][], endings?, repeats? } ↓ songToProgression() Progression { chords: ChordSlot[], chartLayout: ChartLayout } ↓ ChordChart component CSS Grid 描画

Design Decisions

  • chords: ChordSlot[] フラット配列 = ロジックのsingle source of truth
  • chartLayout?: ChartLayout = 表示用メタデータ（chords[] へのインデックス参照）
  • ユーザー作成進行（layoutなし）→ deriveChartLayout() で1コード/小節に自動生成
  • beatWidths: フレックス比率 — ChordChart で flex: beats, BPM 再生で (bw/bwSum)*4
  • ChordSlot.lickId / lickHighOctave / lickHighInstance: リック選択・8va（同一インスタンス内オクターブ上）・Hi（ハイポジションインスタンス切替）をコードごとに独立永続化。コード切替時に自動復元、リセットで全クリア。8vaは音域が広すぎてプールに収まらない場合、Hiはインスタンスが1つしかないポジションでは自動disabled
  • ChordSlot.lickBeatOffset: リックオーバーフロー分割時のビートオフセット。0 = 先頭コード (originator)、>0 = 継続コード、undefined = 分割なし。リック拍数 > コード拍数の場合に sliceLick() で分割し、後続コードに連鎖割当。先頭コードクリア時は全継続コードも連動クリア。ii-V リックも通常リックも同じロジック。

chartLayout Preservation on Edit

以前はコードの追加・削除・編集時に chartLayout を破棄（undefined にセット）していたため、 インポートした曲の小節構造が失われていた。現在は編集操作ごとに chartLayout を差分更新して構造を保持する。

  • 小節追加: 「+ 小節」ボタンで空小節を追加し、即座にビートグリッドの挿入モードに入る。insertEmptyMeasure(layout, afterChordIdx?)
  • コード挿入: ビートグリッドの「+」をクリックして拍位置を選択 → コード名を入力 → Enter で挿入。insertChordAtBeat() / fillEmptyMeasure()
  • コード削除: removeChordFromLayout(layout, deletedIdx) — 該当コードを小節から除去し、後続の全 chordIndices をデクリメント。小節が空になっても小節は残る（空小節として表示）
  • 空小節削除: 空小節の右端に表示される縦長の × ボタンで明示的に削除
  • コード更新: インデックス不変のため chartLayout は変更不要
  • セクション分割: splitSection(layout, sectionIdx, measureIdx) — 指定小節位置でセクションを2つに分割。新セクションには自動ラベル (A-Z) を付与
  • セクション結合: mergeSections(layout, sectionIdx) — 隣接する2セクションを結合。最初のセクションのラベル・リピート設定を保持
  • リピート設定: section.repeats を undefined ↔ 1 でトグル。BPM再生時にセクション全体が2回演奏される

Beat-Position Chord Insertion

編集モードでは ChordChart が4列のビートグリッドを表示し、小節内の特定拍位置にコードを挿入できる。 空き拍に + マーカーが表示され、クリックでその拍位置にコードを挿入する。

  • getMeasureBeatInfo(layout, chordIdx) — 指定コードが属する小節の占有拍数と総拍数を返す
  • computeInsertFlatIndex(layout, referenceIdx, beat) — 小節内の拍位置からフラット配列上のスプライス位置を算出。既に占有されている拍位置には null を返す
  • insertChordAtBeat(layout, referenceIdx, beat, newFlatIdx) — 指定拍位置に新コードを挿入し、beatWidths を比率正規化（合計4拍/小節）して ChartLayout を返す
ビートグリッド表示

編集モード時、ChordChart の各小節は4列グリッドに分割される。 コードが占有する拍はコード名を表示し、空き拍には + ボタンを配置。 beatWidths は常に合計4拍に正規化される。

リック割り当てインジケーター

通常表示モード（非編集時）で、各小節にリック割り当て状況を視覚表示:

  • リックID: コード記号の直下に 8px モノスペースで表示 (色: #FF6B9D)
  • カラーバー: 小節全幅の 4px バー (小節単位で描画)
  • Originator (リック先頭コード): リックID全文 + 不透明ピンクバー (#FF6B9D80)
  • Continuation (オーバーフロー後続コード): → + リックID + 半透明ピンクバー (#FF6B9D50)
  • クリックでコード選択可能
  • リック未割り当て / 編集モード → 非表示

判定: isLickOriginator(chord) (lickPlayback.ts) で originator / continuation を区別。

Section Structure — Symbol Buttons

編集モードでコードの1拍目をクリックすると、ツールバーに記号ボタン行が表示される。 ボタンを押すとその小節にマーカーをスタンプし、必要に応じてセクション分割/結合が自動実行される。 ChordChart 自体は表示専用（リピートドット・ボルタ括弧は自動描画）。

セクション名入力 + 記号ボタン4種:

  • Sec: (セクション名): セクション先頭の小節ではラベルを直接編集。それ以外の小節でラベルを入力するとそこでセクションを自動分割し新セクション名を設定。先頭小節のラベルを空にすると前セクションに結合。splitSection() / mergeSections() / renameSection()
  • |: (リピート開始): 選択小節がセクション先頭ならリピートをトグル。先頭でなければセクションを分割し新セクションにリピートを設定。splitSection()
  • :| (リピート終了): 選択小節がセクション末尾ならリピートをトグル。末尾でなければセクションを分割し前半にリピートを設定。splitSection()
  • 1. (1ボルタ): リピートONのセクション内で、選択小節からボルタ1括弧を設定（残り小節は全て1括弧に入る）。2括弧は別途「2.」で設定。既に1ボルタ内の小節をクリックすると解除。splitEndings() / removeEndings()
  • 2. (2ボルタ): エンディング内の小節をクリックし、その小節から2番カッコを開始。1括弧のみの状態からでも分割可能。adjustEndingSplit()

ボタンの点灯状態: 選択中の小節に既にそのマーカーがある場合は色付き（|:/:| = 黄, 1. = 青, 2. = 緑）。 エンディング内の小節では |:/:| は無効。リピートなしでは 1. は無効。エンディングなしでは 2. は無効。

Phrase Path Rendering

PhrasePath.tsx — Catmull-Rom スプラインを Cubic Bezier に変換して SVG <path> で描画。

  • グラデーション: per-beat (#FFA0B0 → #BBA0FF)
  • ノート形状:
    • CT = 塗り潰し円
    • アプローチ = ダイヤモンド
    • スケールトーン = 枠円（ストロークのみ）
  • 拍番号: 各ノートの横に表示
  • アニメーション: 各ノートのフェードインは beatStart ベースのタイミング（リズムに忠実な表示）
  • 視認性改善:
    • Crowding-aware routing: 各セグメントの中点近接度を距離減衰で計測し、密集エリアの線を垂直方向にファンアウト（最大3層）
    • タンジェント抑制: 短セグメント（同フレット弦移動等）は Catmull-Rom タンジェントをセグメント長比例で減衰させ、ほぼ直線に
    • 折り返しbow抑制: 近接フレット間の方向反転でのbow量をセグメント長に比例してスケール
    • 再訪問マーカー: 75%縮小 + 次ノート方向に6pxオフセット
    • ストローク幅テーパリング: 始点2.2px → 終点1.4pxで時間方向を明示
    • 開始ノート強調: マーカー1.2倍 + 半透明リング
    • コード品質変化でのライン分割は廃止: 旧 chordChangeBeat フィールドは GeneratedPhrase から削除済み。リック拍数がコード拍数を超える場合は sliceLick() でコード拍境界で分割され、各コードに独立した GeneratedPhrase として割り当てられる

機能

  • マルチ小節リック: _N サフィックスで N小節 = 1リック（最大3小節）
  • ソース情報: アーティスト名をファイル名から抽出し "source" フィールドに記録

  • 休符検出: ノート間ギャップを自動検出し { "rest": true } イベントとして記録（先頭・末尾含む）
  • 三連符対応: 24グリッドにより 2/3拍, 1/3拍 等を正確に量子化
  • 重複排除: ピッチ列+リズムが一致するリックは追加しない

ピッチ→指板マッピング

  1. PoolNote[] を absolutePitch (OPEN_MIDI[s] + fret) でインデックス化
  2. リックの各ピッチに移調オフセットを加算
  3. PoolNote から一致するピッチを検索 → 前ノートからの弦距離最小を選択
  4. 候補なし（ポジション外）→ 全弦で fret を計算し最近弦を選択

モード/ポジション自動推定

  • inferModeFromLick(): リックのピッチクラスを各候補モードのスケール音と照合し、一致率最大のモードを選択
  • inferModeCandidates(): 重み付きスコアリング (表拍×2, 長音×2) で最大3件のモード候補を返す。LickPanel のリスト表示に使用
  • findBestPositionForLick(): 各ポジションの absolutePitch セットとリックのピッチを照合し、含有率最大のポジションを選択
  • ユーザーは既存のモード/ポジション選択UIで手動変更可能（自動推定を上書き）

辞典モードの変更

辞典モードはモード/ポジション閲覧専用として機能。指板クリック音も無効化され、GlobalAudioControls（再生バー）も非表示。 ガイドトーンは辞典モードで常時表示され、現モードの 3rd/7th を指板上に表示（次コード情報はなし）。 フレーズ生成・リック練習は練習モード専用。

ii-V Lick Detection

練習モードで ii コード (m7 / m7♭5) が選択された場合、次コードとのルート音程を検査し、ii-V パターンを自動検出。 単コードリックに加えて、2コードにまたがる ii-V リックも LickPanel に表示する。

パターン検出 (detectIiVPattern())

以下の条件が全て満たされた場合に ii-V と判定:

  1. 現コード品質が m7 または m7♭5
  2. 次コードが存在し、品質が 7 系 (dom7)
  3. 現コードの根音 → 次コードの根音が完全5度上 (7半音, P5)

LickPanel 表示

  • 単コードリック一覧の後にセパレーター (── ii-V ──) を挟んで ii-V リックを表示
  • 全リックにタイプバッジを表示（単コード: dom7/min7/maj7/m7♭5、ii-V: ii-V S/ii-V L/ii-V m）
  • ii-V リックのコンタープレビュー・ID・ソース名の表示仕様は単コードリックと同一

再生とBPM自動再生 — リックオーバーフロー分割

  • リック拍数 > コード拍数の場合、sliceLick(lick, offset, beats) でコード拍境界で分割し後続コードに連鎖割当
  • 各コードに ChordSlot.lickBeatOffset (anacrusis=先頭, >anacrusis=継続) を保存。先頭コードクリア時は全継続コードも連動クリア
  • ii-V リックも通常リックも同じロジック。3コード以上の跨ぎにも対応
  • BPM 自動再生では各コードが自身の sliceLick() 結果を独立再生
  • Look-ahead スケジューリング: 次コードの音声 (ストラム + リック + メトロノーム) を Web Audio タイムラインに事前予約。コードチェンジ時の 1-15ms 遅延を解消し、サンプル精度のタイミングを実現。pendingNextRef パターンで cleanup 安全性を確保
  • アウフタクト (anacrusis) 対応: リックにアウフタクト拍数 (LickEntry.anacrusis) がある場合:
    • プレビュー再生: コードストラムをアウフタクト分遅延して鳴らし、メインビートと同期
    • 自動演奏: look-ahead で次コードのアウフタクトを検出し、前のコード再生中にアウフタクト部分を事前再生 + 指板表示
    • カウントイン: 1コード目のリックにアウフタクトがある場合、カウントイン終盤にアウフタクトフレーズを再生
    • オーバーフロー計算は effectiveBeats = lick.beats - anacrusis をベースに行う

永続化

  • リック選択は ChordSlot.lickId + lickBeatOffset + lickAnacrusis で保存
  • オーバーフロー時: 先頭コードに lickBeatOffset: anacrusis、各継続コードに lickBeatOffset: anacrusis + 累積拍数 が設定される (アウフタクトなしの場合は従来通り 0 始まり)
  • コード切替時に自動復元。先頭コードクリア (✕) 時は全継続コードも連動クリア
  • お気に入り: lickFavorites: Set<string> を App.tsx で管理。localStorage キー lickFavorites に JSON 配列として永続化。LickPanel の ★ ボタンでトグル。お気に入りリックは一覧の上位にソートされる
  • 曲ごとの BPM: Progression.bpm?: number フィールドに保存。練習モードで BPM を変更すると現在の進行に自動保存。曲を切り替えると保存済み BPM が復元される。初回起動時は最初の進行の保存 BPM を使用
設計メモ

リックオーバーフロー分割は sliceLick(lick, offset, beats) で汎用的に処理。 旧設計の lickIiVType / lickIiVPart フィールドは lickBeatOffset に統一され、 ii-V リックと通常リックを区別なく同じロジックで分割・割当・再生する。
